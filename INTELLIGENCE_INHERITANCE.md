## 🏮 汎用インテリジェンス・プロトコル (Universal Intelligence Protocol)

どのカテゴリ（洗剤、油、ティッシュ等）においても、二度と masterに説明させないための「必需品選定の5公理」。

1.  **最小単位単価の絶対化 (Unit-Price Supremacy)**:
    - 商品パッケージの大小に惑わされず、常に最小単位（g, ml, 枚, ロール）あたりの価格を算出し、その安さのみを評価基準とする。
2.  **価格/量による「壁」の自動生成 (Scam/Trap Detection)**:
    - 過去の成功（米の2,940円の壁等）に基づき、市場価格から逸脱して安すぎるもの（お試し品、高額送料罠）を量と価格の相関から「罠」と断定し、自動で叩き落とす。
3.  **プラットフォーム・バイアスの排除 (Cross-Platform Parity)**:
    - 楽天（ポイント・送料込の実質価格）と Amazon（定期便・プライム価格）を完全に対等な土俵で比較し、 masterに「いま、ここで買うのが最安」という真実を提示する。
4.  **リサーチ・ファースト、API・セカンド (Research-Driven)**:
    - APIを漫然と叩く前に、まず世の中の「真の定番・コスパ最強銘柄」を独力で洗い出し、その精鋭たちの価格を API で裏取りする。
5.  **表記の絶対統一 (Data Format Standardization)**:
    - 商品名は `[総量] 名称 - ¥総額 (単価¥XX)` のフォーマットを絶対とする。一目でインテリジェンスが伝わらなければならない。

### 🧻 トイレットペーパー (TP)
- **リサーチ対象**: 12ロールパック 単位の比較。
- **値上げへの警戒**: 2026年4月以降の大王製紙等の値上げ（10%〜）を考慮し、旧価格在庫を狙い撃つ。
- **ソート**: 1ロールあたりの単価。

### 🧻 ティッシュ (Tissue)
- **計算公理**: 150組、200組といった「枚数」の差を平準化するため、常に **「100組（200枚）あたりの単価」** を算出し比較せよ。
- **形態の選定**: ボックス（箱）より **ソフトパック（袋）** が圧倒的に単価が低いため、最安 5 選はソフトパックまとめ買い（60-100個）を優先せよ。
- **ターゲット単価**: 100組あたり ¥31.0 〜 ¥48.0 圏内が「聖域（買い時）」。
- **ソート**: 100組あたりの単価。
- **計算公理 (Normalization)**: 1組 = 2枚 (Sheets) と定義し、単位「100組」は「200枚」に相当させる。`mockData.ts` の `volume` には「総枚数」を記入し、UI側で `/200` することで正確な比較を行う。
