## 🚨 [PROTOCOL 0] セキュリティ・絶対遵守事項

1.  **`.env` の秘匿 (Confidentiality)**:
    - `RAKUTEN_APP_ID`, `RAKUTEN_ACCESS_KEY`, `KEEPA_API_KEY`, `GEMINI_API_KEY` 等の値を、**会話ログ、外部出力、GitHub上のファイル内に1文字たりとも記述してはならない。**
    - 漏洩は masterの損失であり、プロジェクトの死を意味する。
2.  **アフィリエイトIDの死守**:
    - AmazonのアソシエイトIDは常に `mangaanimeosu-22` を使用すること。
    - 楽天のアフィリエイトIDは常に `.env` から取得したもの（`RAKUTEN_AFFILIATE_ID`）を使用すること。

## ⚠️ 失敗と改善のリサーチ (Post-Mortem & Failures)

### 1. エージェント・クラッシュ (Day 6)
- **事象**: 楽天APIやGoogle RSSが重い際、エージェントが返答を待機しすぎてハング（Agent execution terminated）。
- **反省**: 外部依存のある全 axios コールにタイムアウトを設定していなかった。
- **改善**: `timeout: 10000` を全サーバーコードに適用。

### 2. PowerShell コマンドチェーンの誤解 (Day 6)
- **事象**: `git push && git commit` 等の `&&` が PowerShell 5.1 環境で動作せず、デプロイが1時間停滞した。
- **反省**: 実行環境（PowerShell）の文法チェックを怠った。
- **改善**: コマンド区切りには `;` を使用、または逐一実行し、ステータスを毎回確認する。

### 3. 書き込みプロセスのハング (Day 6)
- **事象**: `/tmp/` への巨大なファイル書き込みでツールがフリーズ。
- **反省**: 複雑なスクリプトをファイル経由で実行しようとした。
- **改善**: Node.js ワンライナー (`node -e "..."`) を積極的に活用し、ファイル生成を避ける。
